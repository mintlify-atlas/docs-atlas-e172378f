KeyForge shows you the real mathematics behind your password's strength — not a subjective score, but Shannon entropy in bits, a crack-time estimate calibrated to modern hardware, and a per-character visual breakdown.

## What is entropy?

Entropy measures the size of the search space an attacker must exhaust. For a password drawn uniformly at random from a pool of `N` distinct characters, every position contributes `log₂(N)` bits of entropy. A password of `L` characters therefore has:

```
bits = L × log₂(N)
```

Doubling the length adds `log₂(N)` bits per extra character. Doubling the pool size adds only 1 bit per character — which is why length dominates strength.

## Entropy formulas by mode

Each generation mode has a different entropy model in `entropy.js`:

<Tabs>
  <Tab title="Password">
    ```javascript entropy.js
    function calcPasswordEntropy(length, poolSize) {
      if (length <= 0 || poolSize <= 1) return 0;
      return length * Math.log2(poolSize);
    }
    ```

    `poolSize` is the count of characters remaining after applying enabled classes, ambiguous-character exclusion, and custom exclusions — computed by `getPoolSize()`.
  </Tab>
  <Tab title="Passphrase">
    ```javascript entropy.js
    function calcPassphraseEntropy(wordCount, wordListSize = 200, hasNum = true, hasSym = true) {
      let entropy = wordCount * Math.log2(wordListSize);
      if (hasNum) entropy += Math.log2(90);   // 2-digit number: 10–99
      if (hasSym) entropy += Math.log2(7);    // symbol from pool of 7
      return entropy;
    }
    ```

    Each word is chosen from a 200-word list (`log₂(200) ≈ 7.6 bits`). The appended 2-digit number contributes `log₂(90) ≈ 6.5 bits`, and the appended symbol from `!@#$%&*` contributes `log₂(7) ≈ 2.8 bits`.
  </Tab>
  <Tab title="PIN">
    ```javascript entropy.js
    function calcPinEntropy(length) {
      return length * Math.log2(10);
    }
    ```

    Every digit position has 10 possible values, so each contributes `log₂(10) ≈ 3.32 bits`. A 6-digit PIN has ≈ 19.9 bits of raw entropy.
  </Tab>
  <Tab title="Phonetic">
    ```javascript entropy.js
    function calcPhoneticEntropy(syllableCount, entropyLevel) {
      // Syllable space: ~39 onsets × 10 vowels × 17 codas ≈ 6630 + no-coda variants
      const sylSpace = 39 * 10 * 17 + 39 * 10; // rough count
      let entropy = syllableCount * Math.log2(sylSpace);
      // Injected entropy chars
      const injectedChars = { light: 1, medium: 2, heavy: 3 }[entropyLevel] || 2;
      const injections    = syllableCount - 1 + 1; // between + end
      const symPool = 11; // SYMS length
      const digPool = 8;
      entropy += injections * injectedChars * Math.log2(symPool + digPool);
      return entropy;
    }
    ```

    Each syllable is drawn from a space of `39 × 10 × 17 + 39 × 10 = 7,020` combinations (39 onsets, 10 vowels, 17 codas, plus no-coda variants). Injected entropy characters are drawn from a combined symbol + digit pool of 19 (`symPool=11`, `digPool=8`).
  </Tab>
</Tabs>

## Strength levels

`getStrengthLevel()` maps an entropy value to one of six labels defined in `STRENGTH_LEVELS`:

```javascript entropy.js
const STRENGTH_LEVELS = [
  { min: 0,   max: 35,       label: 'Very Weak',   css: 'var(--s0)', pct: 10  },
  { min: 36,  max: 55,       label: 'Weak',        css: 'var(--s1)', pct: 28  },
  { min: 56,  max: 72,       label: 'Fair',        css: 'var(--s2)', pct: 50  },
  { min: 73,  max: 94,       label: 'Strong',      css: 'var(--s3)', pct: 72  },
  { min: 95,  max: 127,      label: 'Very Strong', css: 'var(--s4)', pct: 88  },
  { min: 128, max: Infinity, label: 'Fortress',    css: 'var(--s5)', pct: 100 },
];
```

| Label | Min bits | Max bits | Strength bar |
|-------|----------|----------|--------------|
| Very Weak | 0 | 35 | 10% |
| Weak | 36 | 55 | 28% |
| Fair | 56 | 72 | 50% |
| Strong | 73 | 94 | 72% |
| Very Strong | 95 | 127 | 88% |
| Fortress | 128 | ∞ | 100% |

The boundary at 128 bits corresponds to the minimum recommended by NIST for long-term secrets and matches the key size of AES-128.

## Crack time estimation

`getCrackTime()` converts an entropy value into a human-readable time by simulating an offline attack:

```javascript entropy.js
// Assumes offline attack: 1 trillion guesses/sec (modern GPU farm, raw MD5/SHA)
// For bcrypt/argon2 the real times are orders of magnitude longer.
function getCrackTime(entropyBits) {
  const GPS  = 1e12; // guesses per second
  // Average-case: half the keyspace
  const secs = Math.pow(2, entropyBits - 1) / GPS;

  if (secs < 0.001)           return '< 1 ms';
  if (secs < 1)               return `${(secs * 1000).toFixed(0)} ms`;
  if (secs < 60)              return `${secs.toFixed(1)} sec`;
  if (secs < 3600)            return `${(secs / 60).toFixed(1)} min`;
  if (secs < 86400)           return `${(secs / 3600).toFixed(1)} hrs`;
  if (secs < 2592000)         return `${(secs / 86400).toFixed(1)} days`;
  if (secs < 31536000)        return `${(secs / 2592000).toFixed(1)} months`;
  const yrs = secs / 31536000;
  if (yrs < 1000)             return `${yrs.toFixed(0)} years`;
  if (yrs < 1e6)              return `${(yrs/1e3).toFixed(1)}K years`;
  if (yrs < 1e9)              return `${(yrs/1e6).toFixed(1)}M years`;
  if (yrs < 1e12)             return `${(yrs/1e9).toFixed(1)}B years`;
  if (yrs < 1e15)             return `${(yrs/1e12).toFixed(1)}T years`;
  return '∞ (heat death of universe)';
}
```

The model uses `2^(bits - 1) / 1e12` — half the full keyspace divided by 1 trillion guesses per second — representing the expected average-case search time for an attacker who does not know where in the keyspace the password falls.

<Note>
  The 1 trillion guesses/second rate reflects a modern GPU cluster attacking a fast, unsalted hash (raw MD5 or SHA-1). Against bcrypt at cost factor 12, the same hardware achieves roughly 100,000 guesses/second — making the displayed crack times pessimistic by a factor of around 10 million. Use a proper password hashing algorithm server-side.
</Note>

## Entropy DNA visualization

The DNA bar beneath each generated password gives a per-character breakdown of character class composition. Each character in the password is rendered as a colored cell:

```javascript entropy.js
const DNA_COLORS = {
  upper:  'var(--dna-upper)',   // coral
  lower:  'var(--dna-lower)',   // silver
  digit:  'var(--dna-digit)',   // cyan
  symbol: 'var(--dna-symbol)', // gold
};

function charCategory(c) {
  if (/[A-Z]/.test(c)) return 'upper';
  if (/[a-z]/.test(c)) return 'lower';
  if (/[0-9]/.test(c)) return 'digit';
  return 'symbol';
}
```

<CardGroup cols={4}>
  <Card title="Uppercase" icon="arrow-up">
    Coral. Any character matching `[A-Z]`.
  </Card>
  <Card title="Lowercase" icon="arrow-down">
    Silver. Any character matching `[a-z]`.
  </Card>
  <Card title="Digit" icon="hash">
    Cyan. Any character matching `[0-9]`.
  </Card>
  <Card title="Symbol" icon="asterisk">
    Gold. Anything not matched by the above three categories.
  </Card>
</CardGroup>

Cell heights vary by ±6 px (`8 + Math.floor(Math.random() * 6)`) purely for visual rhythm; height carries no information. Hovering a cell shows a tooltip with the literal character and its class (e.g., `"k" — lower`).

A uniform bar of a single color is a visual signal that you have only one character class enabled — a cue to review your settings.
