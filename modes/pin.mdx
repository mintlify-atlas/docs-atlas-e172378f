The PIN generator produces numeric codes of any length between 4 and 20 digits. Like every KeyForge mode, digit selection uses `crypto.getRandomValues()`. Optional security rules reject sequential runs and well-known PIN patterns before returning a result.

## How it works

<Steps>
  <Step title="Generate digits">
    Each digit position is filled independently using `crypto.getRandomValues()` modulo 10. This gives a uniform distribution across `0–9`.
  </Step>
  <Step title="Check security rules">
    The assembled PIN is tested against each active rule. If any rule is violated, the entire PIN is discarded and a new one is generated — up to 200 attempts.
  </Step>
  <Step title="Return or fall back">
    If a compliant PIN is found within 200 attempts, it is returned. If not — which can occur when **No repeating digits** is on for short PINs — the generator falls back to a plain random PIN without rule enforcement.
  </Step>
</Steps>

## Entropy

A PIN with no restrictions draws from a pool of 10 digits per position:

| Length | Entropy | Combinations |
|---|---|---|
| 4 digits | ~13.3 bits | 10,000 |
| 6 digits | ~20.0 bits | 1,000,000 |
| 8 digits | ~26.6 bits | 100,000,000 |
| 12 digits | ~39.9 bits | 1,000,000,000,000 |
| 20 digits | ~66.4 bits | 10²⁰ |

Each additional digit multiplies the search space by 10 (~3.32 bits).

<Warning>
  PINs are inherently weaker than passwords of the same length because they draw from only 10 possible characters. A 6-digit PIN offers the same entropy as roughly 4 random lowercase letters. Use PINs only where a numeric-only input is required — for everything else, prefer the [Password](modes/password) or [Passphrase](modes/passphrase) modes.
</Warning>

## Settings

### PIN length

<ParamField path="length" type="number" default="6">
  Number of digits in the generated PIN. Range: **4–20**. The strength meter reflects raw entropy at the selected length.
</ParamField>

### Security rules

<ParamField path="noRepeat" type="boolean" default="false">
  Require that the PIN contains at least 7 unique digits. With this rule on, no digit can appear so frequently that fewer than 7 distinct values are present across the full PIN. Disabled by default because it materially reduces entropy at shorter lengths — a 4-digit PIN with all unique digits has only 5,040 combinations instead of 10,000.
</ParamField>

<ParamField path="noSequential" type="boolean" default="true">
  Reject any PIN that contains three or more consecutive digits with a constant difference of ±1 — covering ascending runs like `123`, `456`, `789` and descending runs like `987`, `654`, `321`, as well as any repetition run like `000`, `111`. Enabled by default.
</ParamField>

<ParamField path="noCommon" type="boolean" default="true">
  Reject any PIN that contains one of the following 3-digit substrings anywhere in its length. These are the most-commonly observed PIN fragments in leaked credential databases:

  ```text
  000  111  123  234  345  456  567  678  789
  987  876  765  654  543  432  321  210
  ```

  The check slides a 3-character window across the entire PIN, so a 6-digit PIN like `512345` would be rejected because it contains `123`, `234`, and `345`. Enabled by default.
</ParamField>
