The password generator is the core mode in KeyForge. Every password is built using `crypto.getRandomValues()` — the browser's CSPRNG — and shuffled with a Fisher-Yates algorithm driven by the same entropy source. No `Math.random()` is ever used.

## How it works

<Steps>
  <Step title="Build the character pool">
    KeyForge assembles a pool from the character classes you enable: lowercase `a–z`, uppercase `A–Z`, digits `0–9`, and the symbol set `!@#$%^&*()_+-=[]{}|;:,.<>?/~`. Any characters in the exclude list or the ambiguous set are filtered out before the pool is used.
  </Step>
  <Step title="Guarantee minimums">
    Before random fill begins, KeyForge draws the required minimum count of characters from each enabled class (default: 1 per class). This ensures the output always satisfies your policy requirements.
  </Step>
  <Step title="Force-include custom characters">
    If you specify characters in the Force include field, KeyForge guarantees at least one occurrence of each in the final password, provided they already exist in the pool.
  </Step>
  <Step title="Random fill">
    Remaining positions are filled by randomly sampling from the full pool using `crypto.getRandomValues()` until the target length is reached.
  </Step>
  <Step title="Fisher-Yates shuffle">
    The complete character array — mandatory and random slots together — is shuffled using Fisher-Yates with a CSPRNG random index at every step. This eliminates positional bias.
  </Step>
  <Step title="Anti-pattern validation">
    The shuffled password is tested against every enabled guard. If it fails, the process restarts from step 2. After 200 attempts, KeyForge returns the best available result rather than blocking indefinitely.
  </Step>
</Steps>

<Note>
  The generator retries up to **200 times** to satisfy all active anti-bruteforce guards. If no valid password is found within 200 attempts — which can happen with very short lengths and many guards enabled simultaneously — KeyForge falls back to a plain random password without guard checks, ensuring a result is always returned.
</Note>

## Settings

### Length

<ParamField path="length" type="number" default="16">
  Number of characters in the generated password. Range: **8–128**. The entropy DNA bar and strength meter update in real time as you adjust this slider.
</ParamField>

### Character classes

Enable or disable each class to control which characters appear in your password. At least one class must remain enabled.

<ParamField path="useUpper" type="boolean" default="true">
  Include uppercase letters **A–Z** (26 characters).
</ParamField>

<ParamField path="useLower" type="boolean" default="true">
  Include lowercase letters **a–z** (26 characters).
</ParamField>

<ParamField path="useDigits" type="boolean" default="true">
  Include digits **0–9** (10 characters).
</ParamField>

<ParamField path="useSymbols" type="boolean" default="true">
  Include symbols from the active symbol pool. Default pool: `!@#$%^&*()_+-=[]{}|;:,.<>?/~` (29 characters).
</ParamField>

### Anti-bruteforce guards

<ParamField path="excludeAmbiguous" type="boolean" default="true">
  Remove visually ambiguous characters that are frequently confused when reading or typing: `0`, `O`, `o`, `l`, `1`, `I`, `|`, `` ` ``, `'`, `"`, `\`. Reduces transcription errors for passwords used in non-paste scenarios.
</ParamField>

<ParamField path="noSequential" type="boolean" default="true">
  Reject any password that contains a keyboard-row walk of 3 or more consecutive characters (forward or reverse), such as `qwe`, `asd`, `zxc`, or `123`. Checks all four keyboard rows: `qwertyuiop`, `asdfghjkl`, `zxcvbnm`, and `1234567890`.
</ParamField>

<ParamField path="noRepeating" type="boolean" default="false">
  Reject any password where two adjacent characters are identical, for example `aa` or `11`. Disabled by default because it has a measurable effect on entropy at short lengths.
</ParamField>

<ParamField path="noWeakFragment" type="boolean" default="true">
  Reject any password that contains a substring matching a known weak pattern (case-insensitive). See the full blocked list below.
</ParamField>

#### Blocked weak fragments

The following 4-character (or longer) substrings trigger a rejection when **Block weak fragments** is on:

```text
pass  word  admin  root   user   login  test   demo
temp  qwer  asdf   zxcv   1234   2345   3456   4567
5678  6789  abcd   aaaa   bbbb   cccc   dddd   eeee
hello welco secre  chang  defau  guest  name   email
birth year  month  date   home   work   love
```

### Minimum character counts

Expand the **Minimum character counts** section to guarantee a precise floor for each class. All values default to `1` and accept integers from `0` to `20`.

<ParamField path="minUpper" type="number" default="1">
  Minimum number of uppercase letters that must appear in the password.
</ParamField>

<ParamField path="minLower" type="number" default="1">
  Minimum number of lowercase letters that must appear in the password.
</ParamField>

<ParamField path="minDigits" type="number" default="1">
  Minimum number of digit characters that must appear in the password.
</ParamField>

<ParamField path="minSymbols" type="number" default="1">
  Minimum number of symbol characters that must appear in the password.
</ParamField>

### Custom character sets

Expand the **Custom character sets** section to override the defaults.

<ParamField path="symbolSet" type="string" default="!@#$%^&*()_+-=[]{}|;:,.<>?/~">
  The full set of symbols available when **Symbols** is enabled. Edit this field to restrict or expand the symbol pool — for example, remove `<>` for systems that treat them as HTML.
</ParamField>

<ParamField path="customExclude" type="string" default="">
  A string of individual characters to remove from every class pool. For example, entering `@#&` prevents those three characters from ever appearing, regardless of which classes are active.
</ParamField>

<ParamField path="customInclude" type="string" default="">
  A string of individual characters that are each guaranteed to appear at least once in the output. Only characters that already exist in the assembled pool (after exclusions) are honoured; characters outside the pool are silently ignored.
</ParamField>

## Keyboard shortcut

<Tip>
  Press **Ctrl+Enter** anywhere on the Password tab to regenerate instantly without reaching for the Generate button.
</Tip>
