KeyForge runs entirely in your browser — no sign-up, no installation, no backend. Open the page and you are ready to generate.

<Note>
  KeyForge has a **Zero Data Policy**. All generation happens locally using `crypto.getRandomValues()`. No passwords, settings, or usage data are ever sent to any server. The only thing stored is your theme preference in `localStorage`.
</Note>

## Generate your first password

<Steps>
  <Step title="Open the app">
    Visit [keyforgegd.vercel.app](https://keyforgegd.vercel.app) in any modern browser. A password is generated automatically on load — you do not need to click anything first.
  </Step>
  <Step title="Choose a generator mode">
    Select one of the four tabs at the top of the card:

    | Tab | What it generates |
    |-----|-------------------|
    | **Password** | Random character-based password, 8–128 characters |
    | **Passphrase** | Human-readable word sequence from a 200-word list |
    | **PIN** | Numeric-only PIN, 4–20 digits |
    | **Phonetic Shield** | Pronounceable syllable-based password with injected entropy |

    The **Password** tab is active by default and is the right starting point for most use cases.
  </Step>
  <Step title="Configure your settings">
    Each mode has its own settings panel below the output field. For the Password mode you can adjust:

    - **Length** — drag the slider from 8 to 128 characters (default: 16)
    - **Character classes** — toggle Uppercase A–Z, Lowercase a–z, Digits 0–9, and Symbols on or off
    - **Anti-bruteforce guards** — options like excluding ambiguous characters (`0O1lI`) and blocking keyboard walks are enabled by default
    - **Minimum character counts** — expand the collapsible section to set a minimum count per class (default: 1 each)
    - **Custom character sets** — expand to customise the symbol pool, exclude specific characters, or force-include specific ones

    Settings update the output in real time as you change them — you do not need to click **Generate** again after each adjustment.
  </Step>
  <Step title="Generate">
    Click the **Generate Password** button to produce a new password with your current settings.

    The output field updates immediately. Below it you will see:

    - A **color-coded DNA bar** — each cell represents one character, colored by type (coral = uppercase, silver = lowercase, cyan = digit, gold = symbol)
    - A **strength label** (Very Weak → Weak → Fair → Strong → Very Strong → Fortress)
    - The **entropy in bits** and an **estimated crack time** at 1 trillion guesses per second
  </Step>
  <Step title="Copy your password">
    Click the **copy icon** (⧉) next to the output field, or click anywhere on the output field itself. A "Copied to clipboard ✓" toast confirms the copy.

    The copy button briefly changes to ✓ and resets after 1.5 seconds.
  </Step>
</Steps>

<Tip>
  Press **Ctrl+Enter** (or **Cmd+Enter** on macOS) at any time to regenerate using the current settings — without reaching for the mouse. The shortcut works on whichever tab is currently active.
</Tip>

## What each mode does

<CardGroup cols={2}>
  <Card title="Password" icon="key" href="modes/password">
    Character-based passwords up to 128 characters. Full control over character classes, minimum counts per class, custom symbol pools, and anti-bruteforce guards including keyboard-walk blocking and weak-fragment detection.
  </Card>
  <Card title="Passphrase" icon="list-ul" href="modes/passphrase">
    Word sequences drawn from a curated 200-word list using `crypto.getRandomValues()`. Choose 3–10 words, pick a separator (dash, dot, space, underscore, hash, or none), and optionally append a 2-digit number and symbol.
  </Card>
  <Card title="PIN" icon="hashtag" href="modes/pin">
    Numeric PINs from 4 to 20 digits. Blocks ascending and descending sequences (e.g. 1234, 9876) and common 3-digit patterns (e.g. 000, 123) by default.
  </Card>
  <Card title="Phonetic Shield" icon="shield-halved" href="modes/phonetic-shield">
    Pronounceable passwords built from consonant-vowel syllable groups with high-entropy characters woven between them. Generates a pronunciation guide so you can read the password aloud — while still resisting dictionary and brute-force attacks.
  </Card>
</CardGroup>

## Anti-bruteforce guards

The Password mode ships with four guards. Two are enabled by default:

| Guard | Default | What it blocks |
|-------|---------|----------------|
| Exclude ambiguous characters | On | Characters that look alike: `0`, `O`, `o`, `l`, `1`, `I`, `\|` |
| Block keyboard walks | On | Sequential runs across keyboard rows: `qwer`, `asdf`, `1234` |
| Block weak fragments | On | Common dictionary strings: `pass`, `admin`, `login`, `qwer`, etc. |
| No consecutive repeats | Off | Adjacent identical characters: `aa`, `11` |

The PIN mode also blocks sequential digit runs and common PIN sub-sequences by default.

<Warning>
  If all anti-bruteforce guards and constraints cannot be satisfied within 200 attempts (for example, because the pool is too small for the requested rules), KeyForge falls back to a random password without the strict checks. Widen your character pool or relax constraints if you see unexpected output.
</Warning>
